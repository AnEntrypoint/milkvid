Features

Everything you need to make visualizer videos

Built on battle-tested open source tech for maximum quality and speed.

🎨
16,000+ Milkdrop presets
Browse the full butterchurn preset library with a searchable thumbnail grid. Every classic and community preset, instantly findable.
⚡
GPU-accelerated rendering
Runs butterchurn in a headless Chromium via WebGL2 + ANGLE. Hits 200+ fps on a modern GPU — 6–7× faster than realtime.
🎬
WebCodecs H.264 encoding
Frames are encoded in-browser using the WebCodecs VideoEncoder API with hardware H.264, streamed directly to Node without any intermediate files.
🔊
Audio-synced visuals
FFT analysis drives the Milkdrop beat detection. Drop in a WAV file and the visualization pulses, morphs, and reacts to every beat.
📦
Zero-friction output
ffmpeg muxes the encoded H.264 stream with your audio into a clean MP4 with +faststart, ready to upload anywhere.
🌐
Browser-based UI
The entire workflow lives in a local web app. No GUI framework, no Electron — just Node.js serving a fast, keyboard-friendly interface.
Architecture

How it works

A tight pipeline from browser to file — no temp video files, no re-encoding.

1

Local HTTP server + Puppeteer launch

Node.js starts a local server and opens a headless Chromium instance via Puppeteer. The browser loads the render page with butterchurn initialized and ready.

Node.jsPuppeteerHTTP server
2

WebGL2 visualization + WebCodecs encoding

Each frame is rendered by butterchurn on a WebGL2 canvas, then captured and passed to the WebCodecs VideoEncoder for hardware H.264 encoding in the browser process.

butterchurnWebGL2WebCodecsH.264
3

Encoded chunks streamed to Node

Encoded H.264 chunks are POSTed directly to the Node server as binary data — no disk writes, no temp files. The server pipes them straight to ffmpeg stdin.

HTTP POSTbinary stream
4

ffmpeg mux → MP4 output

ffmpeg muxes the video stream with the WAV audio into an MKV container, then remuxes to MP4 with +faststart. The output file is ready in your browser for download.

ffmpegMKV → MP4faststart

Getting started

Up and running in minutes

Or clone and run from source with Node.js 18+.

1
Download the binaryPick your platform above and download the executable alongside your presets/ and thumbnails/ folders.
2
Run itDouble-click the binary or run it from your terminal. A browser window opens automatically at localhost:3000.
3
Pick a presetUse the search box to filter through 16,000+ presets. Click one to select it — a preview thumbnail shows the visual style.
4
Drop in audioDrag and drop a .wav file into the audio zone, or click to browse.
5
RenderHit Render. Progress updates in realtime. When done, download your MP4.

Or run from source:

# clone and install
git clone https://github.com/AnEntrypoint/milkvid
cd milkvid
npm install

# start
node server.js
